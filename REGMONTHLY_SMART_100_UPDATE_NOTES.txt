RegMonthly Smart 100 Update Notes
Version/date: 2026-05-05

Files changed:
- scripts/build.py

What changed:
1. Monthly pull window includes a one-day buffer on both sides of the prior month.
   Example: the April monthly run pulls 2026-03-31 through 2026-05-01.
   IRS monthly archive URLs still use the actual target month, so April still points to the April IRS archive.

2. Quarterly filter remains in the build pipeline.
   Any article with the word "quarterly" in title, summary, content, category, source, or URL is removed before output files are written.

3. Power Automate smart endpoint remains:
   docs/raw/items-smart-100.json

4. The smart endpoint uses the same JSON array format as:
   docs/raw/items-array.json

5. The main site/data still receives the full post-scrape dataset after the quarterly exclusion.
   The smart top-100 bank/fintech relevance filter is applied only to docs/raw/items-smart-100.json.

6. Enforcement action articles are now excluded from the Smart 100 endpoint only.
   They are not removed from the main site dataset unless they also contain the separate excluded word "quarterly."

7. Enforcement-action exclusion terms include:
   enforcement action, enforcement actions, civil money penalty, consent order,
   cease and desist, prohibition order, formal agreement, and settlement agreement.

8. The smart endpoint ranks the remaining articles by bank/fintech relevance using source weight, regulatory keywords, bank/fintech keywords, and noise penalties.
   It does not add scoring fields to the JSON objects so Power Automate sees the same object format as items-array.json.

Power Automate URL after push/build:
https://jasonw79118.github.io/regmonthly/raw/items-smart-100.json

Suggested replacement instructions:
- Copy scripts/build.py into the repo at scripts/build.py and overwrite the existing file.
- Remove any old update notes TXT files from prior fix packs if you do not want them committed.
- Commit and push.
- Let the normal monthly build generate docs/raw/items-smart-100.json, or run build.py intentionally only when you are ready for the monthly dataset to regenerate.

Important caution:
- Running scripts/build.py will regenerate the monthly article outputs. If you want to preserve the already-captured monthly dataset, only commit the build.py change and wait for the scheduled monthly run.
